ML Pipeline

From raw data to decision

Every step follows the original research notebook exactly — data integrity preserved end-to-end.

1
Data Loading & Exploration
Load loan_approval_data.csv — 1,000 applicant records, 20 features covering income, credit history, employment, and loan details.
2
Missing Value Imputation
Numerical columns filled with column means via SimpleImputer(strategy='mean'). Categorical columns filled with most-frequent values.
3
Label Encoding
LabelEncoder applied to Education_Level (Graduate=0 / Not Graduate=1) and target Loan_Approved (No=0 / Yes=1).
4
One-Hot Encoding
OneHotEncoder(drop='first') applied to Employment Status, Marital Status, Loan Purpose, Property Area, Gender, and Employer Category — creating 15 binary features.
5
Feature Engineering
Polynomial features added: DTI_Ratio² and Credit_Score². Original DTI and Credit Score then dropped from X — 27 final features.
6
Train / Test Split + Scaling
80/20 split with random_state=42. Features standardised with StandardScaler fit on training data only — no data leakage.
7
Model Training & Evaluation
Three classifiers trained: Logistic Regression, KNN (k=5), Gaussian Naïve Bayes. Each evaluated on precision, recall, F1, accuracy, and confusion matrix.
