### 📐 Evaluation Metrics

> **Accuracy alone is a lie. Here's the full story.**
> Five dedicated notebooks — each metric isolated, explained, and demonstrated.

```
┌─────────────────────────────────────────────────────────────────┐
│  Confusion Matrix  ─►  TP / FP / FN / TN                        │
│        │                                                         │
│        ├──► Accuracy   = (TP+TN) / All         ← overall score  │
│        ├──► Precision  = TP / (TP+FP)          ← quality of +   │
│        ├──► Recall     = TP / (TP+FN)          ← coverage of +  │
│        └──► F1 Score   = 2 × (P×R) / (P+R)    ← balanced mean  │
└─────────────────────────────────────────────────────────────────┘
```

| # | Metric | Why it matters | Notebook |
|:-:|:-------|:---------------|:--------:|
| 6 | ✅ Accuracy | Baseline — but misleading with imbalanced data | [accuracy.ipynb](./Logistic_Regression/Evaluation_Metrices/accuracy.ipynb) |
| 7 | 🎯 Precision | Use when false positives are expensive (spam filter) | [precision.ipynb](./Logistic_Regression/Evaluation_Metrices/precision.ipynb) |
| 8 | 🔁 Recall | Use when false negatives are dangerous (cancer detection) | [recall.ipynb](./Logistic_Regression/Evaluation_Metrices/recall.ipynb) |
| 9 | 📐 F1 Score | Use when you need the precision-recall balance | [f1 score.ipynb](./Logistic_Regression/Evaluation_Metrices/f1%20score.ipynb) |

## 🧠 Key Learnings

<details>
<summary><b>📈 Linear Regression</b></summary>

- Feature scaling is mandatory before regularization — Lasso is scale-sensitive
- L1 (Lasso) drives weak feature coefficients exactly to zero — effective automatic feature selection
- Correlation heatmaps expose multicollinearity before you even train a model
- OLS gives the exact analytical solution; Gradient Descent scales better to large feature sets

</details>

<details>
<summary><b>🔀 Logistic Regression</b></summary>

- Accuracy is deceptive with imbalanced classes — always check Precision, Recall, F1
- In medical models (heart attack), maximizing Recall is more important than Precision — missing a real case is costlier than a false alarm
- The decision boundary threshold (0.5 by default) should be tuned to the business cost of each error type

</details>

<details>
<summary><b>🌿 KNN</b></summary>

- Feature normalization is non-negotiable — KNN is entirely distance-based, so scale matters enormously
- Choosing K: too small → overfit; too large → underfit. Plot error rate curve to find the sweet spot
- Cross-validation gives a much more honest accuracy than a single train-test split

</details>

<details>
<summary><b>🧮 Naive Bayes</b></summary>

- The independence assumption is almost always violated in real data — yet NB still works remarkably well
- Training is near-instant; great baseline model for any classification problem
- Works well even on small datasets where complex models overfit

</details>

<details>
<summary><b>⚙️ Scratch Implementations</b></summary>

- Building OLS from scratch demystifies the normal equation and makes matrix algebra feel real
- Gradient Descent needs careful learning rate selection — too high diverges, too low converges extremely slowly
- Watching the cost function decrease during training is genuinely satisfying

</details>
